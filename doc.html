This scheme library provides ability to use data received in TUIO protocol.

New in scheme-tuio-0.2: path recording

Installation

Simply put the SchemeTuio.ss included in this package into your source directory and require it:

(require "SchemeTuio.ss")

Functions

  • (start-tuio-client ip port) Starts a tuio client listening to specified ip (String) and port (integer)
  • (get-tuio-cursors) Returns a list-of-2Dcur containing all visible cursors
  • (get-tuio-objects) Returns a list-of-2Dobj containing all visible objects

Structures

  • TuioPoint: posX posY
  • 2Dcur: sID posX posY vecX vecY movAcc path
  • 2Dobj: sID symID posX posY angle vecX vecY rotVec movAcc rotAcc path

sID: integer (session ID, unique)
symID: integer (symbol ID, for objects)
posX, posY: float [0,1] (position)
angle: float [0,2PI] (angle)
vecX, vecY: float (velocity vector)
rotVec: float (rotation velocity vector)
movAcc: float (movement acceleration)
rotAcc: float (rotation acceleration)
path: list-of-TuioPoint

Use

Tuio Client needs to be started first with host ip and port:

(start-tuio-client "127.0.0.1" 3333)

If you are using this library in a world/universe application, remember to call this before your big-bang call.
Once the Client has been started, you can use the above two functions at any time to get the latest list of visible cursors/objects.

Notes

For example use and further explanations, please refer to the demo application (TuioDemo.scm).
Client thread needs to be killed manually. A simple method is shown in demo application.
